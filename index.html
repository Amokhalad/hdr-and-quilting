Project 1: High Dynamic Range

Background

Modern cameras are unable to capture the full dynamic range of commonly encountered real-world scenes. In some scenes, even the best possible photograph will be partially under or over-exposed. Researchers and photographers commonly get around this limitation by combining information from multiple exposures of the same scene. In this project, we will show how we wrote software to automatically combine multiple exposures into a single high dynamic range radiance map, and then convert this radiance map to an image suitable for display through tone mapping.

There are two major parts to this project:

  1. Recovering a radiance map from a collection of images
  1. Converting this radiance map into a display image

For the first part we will have to recover the inverse of the function mapping exposure to pixel value, g.

The the second part we will start by implementing your a global tone mapping operator, and you show how we improved it to a local tone mapping procedure.

The two relevant papers are Debevec and Malik 1997 and Durand 2002.

Part 1: Recovering a radiance map

Following Debevec and Malik's method, we recover the radiance map in two key steps: solving for the camera response function ggg﻿ and combining multiple exposures to create the HDR radiance map.

Camera Response Function Recovery

We solve for ggg﻿ using a system of equations that considers:

  1. Data-fitting terms: g(Zij)=ln⁡(Ei)+ln⁡(Δtj)g(Z_{ij}) = \ln(E_{i}) + \ln(Δt_{j})g(Zij​)=ln(Ei​)+ln(Δtj​)﻿
  1. Smoothness constraint: ggg﻿ should be smooth, enforced by minimizing its second derivative
  1. Weighting function: w(z)=z for z≤127.5;  w(z)=255−z for z>127.5w(z) = z \text{ for } z ≤ 127.5; \; w(z) = 255-z \text{ for } z > 127.5w(z)=z for z≤127.5;w(z)=255−z for z>127.5﻿

The system is solved separately for each color channel using least squares optimization with SVD. Figure 1 shows the recovered g functions for each channel, demonstrating the nonlinear relationship between pixel values and log exposure.

HDR Radiance Map Construction

Using the recovered g functions, we compute the radiance map by:

  1. Converting pixel values to log radiance: ln⁡(Ei)=g(Zij)−ln⁡(Δtj)\ln(E_i) = g(Z_{ij}) - \ln(Δt_j)ln(Ei​)=g(Zij​)−ln(Δtj​)﻿
  1. Combining multiple exposures using weighted average based on pixel reliability
  1. Converting from log space to linear space: Ei=exp⁡(ln⁡(Ei))E_i = \exp(\ln(E_i))Ei​=exp(ln(Ei​))﻿

Figure 2 shows the resulting radiance map visualized in log space. The high dynamic range is evident from the detailed preservation in both bright areas (windows, direct light) and dark regions (shadows, indoor areas).

The recovered radiance map successfully captures the full dynamic range of the scene, though raw values require tone mapping for display on standard devices.

Part 2: Tone Mapping

We implemented three tone mapping approaches to convert the HDR radiance map to displayable images:

Global Operators

  1. Global Scale: Simple linear scaling of radiance values to [0,1] range. While straightforward, this approach often loses detail in extreme brightness regions.
  1. Global Simple: Implements log⁡(L)/max⁡(log⁡(L))\log(L)/\max(\log(L))log(L)/max(log(L))﻿ transformation. This better preserves details across the dynamic range compared to linear scaling, but can appear flat.

Local Operator: Durand's Bilateral Filter

Our main implementation follows Durand's bilateral filtering approach:

  1. Decompose log-intensity into base and detail layers:
    • Base layer (B): Apply bilateral filter to capture large-scale variations
    • Detail layer (D): Subtract base from log-intensity
  1. Compress dynamic range of base layer while preserving detail:
    • Scale base layer to desired dynamic range (typically 2-8 stops)
    • Recombine with unmodified detail layer
    • Apply gamma correction for display
  1. Preserve color by:
    • Computing chrominance ratios (R/I, G/I, B/I)
    • Applying tone mapping to intensity only
    • Reconstructing RGB using mapped intensity and original ratios

This local approach better preserves local contrast and detail while compressing the overall dynamic range. Figure 3 compares results across all three methods, while Figure 4 demonstrates the decomposition into base and detail layers.

Project 2: Image Quilting for Texture Synthesis and Transfer

Background

The goal of this project was to implement the image quilting algorithm for texture synthesis and transfer, based on the seminal SIGGRAPH 2001 paper by Efros and Freeman. The project explores techniques for creating larger textures from small sample images and transferring texture characteristics between images.

Below we implement three different techniques for texture synthesis, gradually getting more complex.

  1. Random Texture Synthesis: The first is as simple as randomly sampling patches from an input texture to create an output image.
  1. Overlapping Patches Synthesis: This technique improves upon the random sampling by introducing patch overlapping and cost-based patch selection.
  1. Seam Finding: This method intrudcues advanced path blending using seam finding by computing a minimum-cost paths (we can use Dijstra algorithm) to find optimal seam routs.

Implemented Techniques

1. Random Texture Synthesis

The first approach, quilt_random(), randomly samples square patches from the input texture to create an output image. This method serves as a baseline, demonstrating the simplest form of texture synthesis.

Key Implementation:

  • Extracts random patches of a specified size
  • Tiles patches to fill the output image
  • Simple but produces visually discontinuous results

2. Overlapping Patches Synthesis

The quilt_simple() method improves upon the random sampling by introducing patch overlapping and cost-based patch selection.

Key Components:

  • SSD Patch Matching: Implemented ssd_patch() to compute the Sum of Squared Differences (SSD) between overlapping regions
  • Patch Selection: choose_sample() selects patches with low SSD costs, introducing controlled randomness via the tolerance parameter
  • Overlapping regions help create more coherent texture transitions

3. Seam Finding

The quilt_cut() method introduces advanced patch blending using seam finding:

  • Computes minimum-cost paths through overlapping regions
  • Uses the cut() function to find optimal seam routes
  • Blends patches along these low-cost paths to reduce visible artifacts

Texture Transfer

The texture_transfer() method extends the quilting algorithm to transfer texture characteristics:

  • Incorporates a correspondence term based on image illuminance
  • Allows guiding the texture synthesis process using a target image's structure
  • Balances patch matching and target image correspondence through a weight parameter
